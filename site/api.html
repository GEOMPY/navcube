    API Reference¶

    This page contains the generated API documentation for geompy_display using mkdocstrings.

    Package Overview¶

    Use these symbols for typical application integration:

    • OCCDisplay: Main viewer wrapper with ViewCube integration.
    • ViewCubeConfig: Configuration model for size, colors, placement, and rendering behavior.
    • ViewCube: Low-level ViewCube builder/manager.

    geompy_display ¶

    geompy-display: Orientation ViewCube for pythonocc-core viewer.

    A Pythonic wrapper around pythonocc-core's viewer with an always-on orientation ViewCube featuring zoom-adaptive stroke rendering.

    Attributes:

    Name Type Description
    __version__ str

    Package version.

    Examples:

    Basic usage with default ViewCube configuration:

    >>> from geompy_display import OCCDisplay, ViewCubeConfig
>>> display = OCCDisplay(title="My App")
>>> @display.on_ready
... def populate(d):
...     from OCC.Core.BRepPrimAPI import BRepPrimAPI_MakeBox
...     from OCC.Core.AIS import AIS_Shape
...     d.context.Display(AIS_Shape(BRepPrimAPI_MakeBox(50,30,20).Shape()), True)
>>> display.start()

    Customizing ViewCube position and size:

    >>> cfg = ViewCubeConfig(cube_size=30, position="top-left")
>>> display = OCCDisplay(title="My App", viewcube=cfg)

    Classes¶

    ViewCubeConfig dataclass ¶

    Configuration for ViewCube appearance, position, and rendering.

    This dataclass controls all visual and layout parameters of the orientation ViewCube. Includes validation in post_init() and supports dynamic updates via update(). Derived fields (char_height, char_gap) are computed from base parameters and updated automatically on any change.

    Attributes:

    Name Type Description
    cube_size float

    Size of the cube in world units. Defaults to 25.0.
    chamfer_r float

    Radius of chamfered (beveled) corners in world units. Defaults to 2.0. Must be < cube_size / 4.
    position str

    Corner position. One of POSITIONS. Defaults to "top-right".
    padding float

    Gap between cube and viewport edge in world units. Defaults to 5.0.
    silver Tuple[float, float, float]

    RGB color of cube faces, each channel in [0..1]. Defaults to (0.75, 0.75, 0.78).
    label_color Tuple[float, float, float]

    RGB color of text labels, each channel in [0..1]. Defaults to (0.0, 0.0, 0.0) for black.
    side_margin float

    Fraction of flat face reserved as empty space on each side. Defaults to 0.15 (15% margin each side).
    char_aspect float

    Character width ratio relative to height. Defaults to 0.60.
    line_width int

    Glyph stroke width in pixels. Defaults to 1. Keep at 1; zoom-adaptive rendering handles thickness.
    char_height float

    Computed character height (derived field, read-only).
    char_gap float

    Computed gap between characters (derived field, read-only).

    Raises:

    Type Description
    ValueError

    If parameters fail validation (e.g., invalid colors, cube_size <= 0, invalid position).

    ViewCube ¶

    Builder and manager for 3D orientation cube with labeled faces.

    Constructs a chamfered cube with labeled faces (FRONT, BACK, LEFT, RIGHT, TOP, BOTTOM) positioned in a screen corner. Handles scaling to world-space, text rendering on each face, and zoom-adaptive line width for visibility at any zoom level.

    Attributes:

    Name Type Description
    cfg ViewCubeConfig

    ViewCubeConfig controlling appearance and position.
    _ais_cube AIS_Shape | None

    AIS_Shape for the cube body (set by show()).
    _label_ais list[AIS_Shape]

    List of AIS_Shape objects for face labels (set by show()).
    _last_width int | None

    Cache of last applied line width for optimization.
    Functions¶
    show(ctx, viewer_widget) ¶

    Display ViewCube in the viewer at the configured corner position.

    Builds the cube geometry, positions it in world space, creates and positions face labels, and displays both in the OCC context. The cube is sized proportional to the viewport dimensions for responsive layout.

    Parameters:

    Name Type Description Default
    ctx AIS_InteractiveContext

    OCC AIS_InteractiveContext where the cube is displayed.

    		 required
    viewer_widget qtViewer3d

    qtViewer3d widget for reading viewport dimensions and view information.

    		 required
    Side Effects

    Updates self._ais_cube and self._label_ais with display objects. Increases context display count.

    hide(ctx) ¶

    Erases both the cube body and face labels from the OCC context without destroying the geometry objects.

    geompy_display.viewcube_config ¶

    ViewCube configuration dataclass for appearance and positioning.

    Provides ViewCubeConfig, a dataclass that controls all aspects of the 3D orientation cube rendering—size, position, colors, text layout, and rendering parameters. Settings support dynamic updates via the update() method.

    Attributes:

    Name Type Description
    POSITIONS tuple[str, ...]

    Valid corner position values.

    Examples:

    Create a custom ViewCube configuration:

    >>> from geompy_display import OCCDisplay, ViewCubeConfig
>>> cfg = ViewCubeConfig(
...     cube_size=30,
...     position="top-left",   # or "top-right", "bottom-left", "bottom-right"
...     silver=(0.7, 0.7, 0.75),
...     padding=10,
... )
>>> display = OCCDisplay(viewcube=cfg)

    Update configuration after creation:

    >>> cfg.update(cube_size=40, position="bottom-right")

    geompy_display.viewcube ¶

    ViewCube builder and manager for OCC viewer orientation cube.

    Manages the construction, positioning, and rendering of a 3D orientation cube with labeled faces (FRONT, BACK, LEFT, RIGHT, TOP, BOTTOM). Provides automatic zoom-adaptive line width rendering and screen-corner positioning.

    Core Functions

    _translate: Translate a shape in 3D space. _build_text_edges: Build text geometry from stroke glyphs. _face_defs: Define label positions and orientations. _position_offset: Compute corner offset from position string. _corner_offset: Compute world-space cube position and size.

    Main Class

    ViewCube: Builds and manages the cube visualization.

    geompy_display.glyphs ¶

    Hershey-style stroke font glyphs for 3D text rendering.

    Provides a dictionary of character glyphs, each defined as a list of polyline strokes in a normalized 0..1 × 0..1 bounding box. Designed for rendering text in 3D space by scaling to world coordinates.

    Attributes:

    Name Type Description
    GLYPHS dict[str, list[list[tuple[float, float]]]]

    Character glyph definitions. Each key is an uppercase letter or space. Each value is a list of strokes, where each stroke is a list of (x, y) normalized coordinates.

    Examples:

    >>> from geompy_display.glyphs import GLYPHS
>>> # Glyph 'A' has two strokes
>>> len(GLYPHS['A'])
2
>>> # First stroke is the outline
>>> GLYPHS['A'][0]
[(0, 0), (0.5, 1), (1, 0)]
